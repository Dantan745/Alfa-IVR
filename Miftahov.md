**[ ПРОБЛЕМНОЕ ПОЛЕ ]**

Зачем нужен проект:
Проект нужен для разработки нового и современного приложение на Android по взаимодействию с цифровым рублем. Основные причины создания проекта:

1. Формирование доступной и защищенной системы платежей на основе цифрового рубля.
2. Создание логичной системы для финансовых операций, с возможностью отслеживать все операции и транзакции.
3. Бесплатные операции для граждан. Для бизнеса вводятся комиссии, которые варьируются в зависимости от величины самой компании.


Какие проблемы данный проект решает:

Проблемы граждан:
1. Отсутствие предсказаний рисков для граждан по различным вкладам.
2. Рекомендательные системы недостаточно четко прогнозируют интересы пользователей (граждан).
3. Неудобство управления личными финансами и отсутствие простой автоматизации платежей. Например, автоматическое выполнение регулярных покупок.

Проблемы бизнеса:
1. Отсутствие аналитики действий клиентов и задержек платежей на основе данных.
2. Сложности с выявлением аномалий. Например, выявление мошенников и нелегальных операций.
3. Отсутствие автоматического контроля оплаты по договорам.

Проблемы государства:
1. Нет автоматизированной поддержки сервиса на основе данных. Например, социальные выплаты или субсидии приходят с задержкой, а не сразу
2. Недостаточная прозрачность потока бюджетных средств. Не все операции прозрачны.


**[ ОБРАЗ ПРОДУКТА ]**

Образ продукта (MVP)

Название продукта: Alfa Кошелек ЦР

Тип продукта: Приложение на Android

Масштаб: Приложение рассчитано для граждан и бизнесов, которые производят свою деятельность на территории РФ.

Функционал с точки зрения машинного обучения:

- Анализ потока операций пользователей для выявления аномалий, включая классификацию транзакций.
- Персонализированные рекомендации пользователю, основанные на анализе данных пользователя в приложении.
- Автоматизация процессов таких как: социальные выплаты конкретным лицам, оплаты по договорам, автоматические оплаты регулярных покупок.
- Систематизированная поддержка пользователя. Система уведомляет пользователей о рисках, важных изменениях и объявлениях, а также предупреждает о потенциальных проблемах, таких как взлом.


Демонстрационный сценарий, действия пользователя:
- Вход в приложение, происходит обязательная регистрация.
- Настройка автоматический платежей и переводов.
- Совершение переводов, выплат и оплат пользователем цифровым рублем; анализ действий пользователя, и система делает рекомендации пользователю.
- Просмотр истории своих действий в приложении.
- Системная отправка пользователю о рекомендациях, изменениях и рисках со стороны его счета, что делает простое взаимодействие пользователя с приложением. Уведомления приходят в реальном времени.

Внутри этих действий и процессов происходит работа ML-моделей для анализа и прогнозирования. Данные всех пользователей защищены.
