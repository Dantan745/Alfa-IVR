Зачем нужен проект:
Проект нужен для разработки нового и современного приложение на Android по взаимодействию с цифровым рублем. Основные причины создания проекта:

1. Формирование доступной и защищенной системы платежей на основе цифрового рубля.
2. Создание логичной системы для финансовых операций, с возможностью отслеживать все операции и транзакции.
3. Бесплатные операции для граждан. Для бизнеса вводятся комиссии, которые варьируются в зависимости от величины самой компании.


Какие проблемы данный проект решает:

Какие проблемы он решает
1. Большие комиссии при переводах между банками и через сторонние сервисы.
    Цифровой рубль позволяет сделать это без комиссий.

2. Риски мошенничества при использовании карт.
    Это не такая же система, как карты, цифровой рубль позволяет видеть весь цифровой след,         
    отсутствие обналичивания закрепляет рубль в системе, не дает отмыть "грязные" деньги.

3. Неудобство управления личными финансами и отсутствие простой автоматизации платежей.
    В нашем приложении будет реализована система контрактов для решения этой проблемы.

4. Зависимость национальной платежной системы от зарубежных технологий.
    Цифровой рубль позволяет делать переводы от лица к лицу напрямую в обход всем иностранным    
    инфраструктурам.

5. Недостаточная прозрачность движения бюджетных средств. Не все операции, которые происходят видны.
    Цифровой рубль оставляет после операций четкий след, по которому видно весь путь денежных   
    средств.

Что будет создано (основные компоненты):

Интерфейс пользователя (Frontend):
Несколько связанных страниц с удобным дизайном. Основные экраны:

 "Обзор кошелька" - баланс пользователя, список действующих контрактов.

 "Конструктор контракта" - кнопка на странице "Обзор кошелька" будет открывать новый экран, где можно будет ввести название, условие контракта, денежную сумму для перевода, лимит для переводов.

 "История операций" - отдельный экран с двумя списками, один - расходы, другой - доходы.

В приложении будет встроен эмулятор для создания контракта - это поможет пользователю разобраться с этой системой, чтоб в дальнейшем при работе с ней избежать ошибок и непонимания механизма системы.

Будет разрабатываться в онлайн сервисе по созданию макетов приложений Figma.
