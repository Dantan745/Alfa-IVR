Название проекта - "Цифровой рубль"

[ ПРОБЛЕМНОЕ ПОЛЕ ]

Зачем нужен проект:
1.Текущие системы для онлайн платежей недостаточно защищены и прозрачны для отслеживания мошенников.
2.Отсутствие стабильной сети не позволяет быть всегда уверенным в способности оплатить через банковское приложение.
3.Сложно вести учет трат, хочется видеть полную картину трат.
4.Отсутствие инструментов на рынке для автоматизации периодических платежей.
5.Сложная навигация и перегруженность интерфейса забирает у пользователя много времени на проведение базовых операций.
6.Непоследовательный и запутанный user flow (Трудоемкие сценарии) из-за которого зачастую раскрывается только малая доля возможностей приложения.
7.Неадаптивный и неинклюзивный дизайн вводят пользователей в заблуждение.

Какие проблемы данный проект решает:

Какие проблемы он решает
1. Большие комиссии при переводах между банками и через сторонние сервисы.
    Цифровой рубль позволяет сделать это без комиссий.

2. Риски мошенничества при использовании карт.
    Это не такая же система, как карты, цифровой рубль позволяет видеть весь цифровой след,         
    отсутствие обналичивания закрепляет рубль в системе, не дает отмыть "грязные" деньги.

3. Неудобство управления личными финансами и отсутствие простой автоматизации платежей.
    В нашем приложении будет реализована система контрактов для решения этой проблемы.

4. Зависимость национальной платежной системы от зарубежных технологий.
    Цифровой рубль позволяет делать переводы от лица к лицу напрямую в обход всем иностранным    
    инфраструктурам.

5. Недостаточная прозрачность движения бюджетных средств. Не все операции, которые происходят видны.
    Цифровой рубль оставляет после операций четкий след, по которому видно весь путь денежных   
    средств.


[ ОБРАЗ ПРОДУКТА ]

Образ продукта 

Название продукта: Alfa Кошелек ЦР.

Тип продукта: Приложение на андроид.

Что будет создано (основные компоненты):

1. Мгновенные P2P-переводы: Перевод средств между пользователями по номеру телефона или ID без комиссий и задержек.

2. Визуальный конструктор смарт-контрактов(frontend): Простой интерфейс, понятный всем категориям пользователей для настройки автоматических платежей (например, регулярные платежи за аренду или подписки) без программирования.

3. Автоматизация бизнес-платежей: Возможность для компаний настраивать автоматические списания с клиентов и выплаты поставщикам по заданным условиям.

4. Детализированная история операций(frontend): Полная лента всех транзакций с расширенным поиском и фильтрами по дате, типу и контрагенту.

5. Работа с данными офлайн: Возможность просматривать историю, готовить платежи и настраивать контракты без интернета с последующей синхронизацией.

6. Массовые выплаты для бизнеса: Инструмент для загрузки CSV-файла со списком получателей для отправки десятков и сотен платежей одной операцией.

7. Управление личными финансовыми лимитами: Настройка лимитов на ежедневные или ежемесячные расходы с уведомлениями об их приближении.

8. Ведение адресной книги: Создание списка частых контрагентов для быстрых переводов.

9. Генерация и сканирование QR-кодов: Упрощённый обмен реквизитами для мгновенных платежей через QR.

10. Встроенная система уведомлений(frontend): Удобная система Push-уведомлений и оповещений внутри приложения о всех важных событиях: переводах, исполнении контрактов, срабатывании лимитов должны быть ко

11. Безопасная аутентификация: Защищённый вход с двухфакторной проверкой и управление сессиями.

12. Панель бизнес-аналитики(frontend): Для компаний — дашборд с наглядными графиками для обработки клиентских транзакций.
